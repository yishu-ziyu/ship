# yishuship: PM 评分标准

## 评分原则

**可机械评估**：每个维度必须能通过规则判断，不依赖"感觉好不好"。
**0-3 分制**：0=缺失 / 1=存在但不合格 / 2=合格 / 3=优秀。
**合格线**：每个阶段总分 ≥ 60%（满分的 60%）才能进入下一阶段。

## 阶段 1：发现（Discover）

**满分 27 分（9 维度 × 3 分）**

| 维度 | 0 分 | 1 分 | 2 分 | 3 分 | 检测方法 |
|------|------|------|------|------|----------|
| **用户画像具体性** | 无用户描述 | "用户"但无角色 | 有角色但无场景 | 角色+场景+技术水平 | 正则匹配：包含"角色"/"场景"/"技术"关键词 + 具体名词 |
| **现有方案描述** | 无 | 提到"现有方案" | 描述了方案但无细节 | 方案+具体工具/流程名 | 包含具体工具名或流程步骤 |
| **问题证据** | 无证据 | "我觉得"/"应该" | 有一个来源 | 2+ 独立来源交叉验证 | 统计来源数：URL/数据引用/用户反馈引用 |
| **竞品数量** | 0 | 1 | 2 | 3+ | 统计竞品表格行数 |
| **竞品分析深度** | 无分析 | 只列名称 | 有方案描述 | 方案+优劣+用户评价 | 每个竞品至少 3 列（方案/优势/劣势） |
| **机会判断** | 无判断 | "值得做"无理由 | 有理由但无证据 | 理由+证据+不做代价 | 包含"值得"/"不值得" + "因为" + 具体证据 |
| **独特优势** | 无 | 提到"我们" | 有优势描述 | 优势+竞品对比 | 包含"我们" + 对比性描述 |
| **频率/严重度** | 无 | 提到其中一个 | 两个都有 | 有量化描述 | 包含频率词（每天/每周/偶尔）+ 严重度词（阻塞/影响/可忍受） |
| **文档完整性** | 无文档 | 有标题无内容 | 内容不全 | 完整模板 | 检查必需章节是否都存在 |

**合格线**：≥ 17 分（满分 27 的 63%）

## SkillOpt 集成格式

每个评分维度可以编码为 SkillOpt 的 benchmark：

```python
# 示例：阶段 1 的 "竞品数量" 维度
def score_competitor_count(output: str) -> float:
    """统计竞品表格行数，返回 0-3 分。"""
    import re
    # 找竞品表格（以 | 竞品 | 开头的表格）
    table_match = re.search(r'\| 竞品.*?\n(\|.*?\n)+', output, re.DOTALL)
    if not table_match:
        return 0.0
    rows = [l for l in table_match.group().strip().split('\n') 
            if l.startswith('|') and '---' not in l]
    count = len(rows) - 1  # 减去表头
    if count >= 3: return 3.0
    if count >= 2: return 2.0
    if count >= 1: return 1.0
    return 0.0
```

### 评分函数设计原则

1. **正则优先**：能用正则匹配的不用 LLM 判断
2. **结构检测**：检查必需章节是否存在、是否非空
3. **数量统计**：统计列表项数、表格行数、引用数
4. **格式验证**：Given/When/Then、P0/P1/P2、DEC-NNNN 等结构化格式
5. **内容质量**：只有内容质量维度（如"分析深度"）才需要 LLM 判断，且必须用独立的 judge model
